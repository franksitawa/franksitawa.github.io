[XTTS-v2](https://github.com/coqui-ai/TTS) from Coqui is one of the most capable open-source TTS systems available today. It supports multilingual synthesis and voice cloning from a short audio sample — all running locally on your hardware.

This guide walks through running XTTS-v2 inside Docker on Ubuntu with CUDA-accelerated GPU inference, which makes speech generation dramatically faster than CPU-only execution.

---

## Prerequisites

Before starting, make sure you have:

- Ubuntu 22.04 or 24.04
- An NVIDIA GPU with drivers installed
- Docker and Docker Compose installed

Verify your GPU is visible to the system:

```bash
nvidia-smi
```

You should see a table showing your GPU name, driver version, and memory usage. If this command fails, install your NVIDIA drivers first before proceeding.

---

## Step 1: Install Docker

```bash
sudo apt update
sudo apt install docker.io -y
sudo systemctl enable docker
sudo systemctl start docker
docker --version
```

---

## Step 2: Install Docker Compose

```bash
sudo apt install docker-compose-plugin -y
docker compose version
```

---

## Step 3: Install the NVIDIA Container Toolkit

By default, Docker containers cannot access your GPU. You can confirm this by running:

```bash
docker info | grep -i runtime
```

If the output only shows `runc` and not `nvidia`, Docker has no GPU support yet.

Add the NVIDIA repository and install the toolkit:

```bash
curl -fsSL https://nvidia.github.io/libnvidia-container/gpgkey | \
  sudo gpg --dearmor -o /usr/share/keyrings/nvidia-container-toolkit-keyring.gpg

curl -s -L \
  https://nvidia.github.io/libnvidia-container/stable/deb/nvidia-container-toolkit.list | \
  sed 's#deb https://#deb [signed-by=/usr/share/keyrings/nvidia-container-toolkit-keyring.gpg] https://#g' | \
  sudo tee /etc/apt/sources.list.d/nvidia-container-toolkit.list

sudo apt update
sudo apt install -y nvidia-container-toolkit
sudo nvidia-ctk runtime configure --runtime=docker
sudo systemctl restart docker
```

Verify the NVIDIA runtime is now registered:

```bash
docker info | grep -i runtime
# Expected: Runtimes: io.containerd.runc.v2 nvidia runc
```

---

## Step 4: Test GPU Access from Docker

Before deploying XTTS, confirm that Docker can see the GPU:

```bash
docker run --rm --gpus all \
  nvidia/cuda:12.2.0-base-ubuntu22.04 \
  nvidia-smi
```

If you see GPU info printed from inside the container, you are ready to proceed.

---

## Step 5: Create the Project Structure

```bash
mkdir xtts && cd xtts
mkdir speakers output
```

Your directory should look like this:

```
xtts/
├── speakers/
└── output/
```

The `speakers/` folder will hold your voice reference WAV files, and `output/` is where generated audio will be written.

---

## Step 6: Create the Docker Compose File

Create `docker-compose.yml` inside the `xtts/` directory:

```yaml
services:
  xtts:
    image: daswer123/xtts-api-server
    container_name: xtts

    ports:
      - "8020:8020"

    volumes:
      - ./speakers:/app/speakers
      - ./output:/app/output
      - xtts_models:/app/models

    deploy:
      resources:
        reservations:
          devices:
            - driver: nvidia
              count: all
              capabilities: [gpu]

    restart: unless-stopped

volumes:
  xtts_models:
```

> **Note:** Do not override the container's `command:` field. The `daswer123/xtts-api-server` image uses a custom entrypoint, and overriding it with `python -m xtts_api_server` causes an `exec: python: not found` error. Let the image use its defaults.

---

## Step 7: Start the Container

```bash
sudo docker compose up -d
sudo docker ps
```

Check that the container is running, then follow the logs:

```bash
sudo docker logs -f xtts
```

The first startup will download the XTTS-v2 model, which may take a few minutes.

---

## Step 8: Add a Voice Sample

Record a clean WAV file of the voice you want to clone — 6 to 30 seconds of clear speech works best. Tools like [Audacity](https://www.audacityteam.org/) or OBS Studio work well on Ubuntu.

Place the file in:

```
speakers/frank.wav
```

Verify the container can see it:

```bash
docker exec -it xtts ls /app/speakers
# frank.wav
```

---

## Step 9: Generate Speech

Open the API docs in your browser:

```
http://localhost:8020/docs
```

Send a request to the `/tts_to_file` or `/tts_to_audio` endpoint with a payload like:

```json
{
  "text": "Welcome to my YouTube channel.",
  "speaker_wav": "frank.wav",
  "language": "en"
}
```

XTTS will clone the voice from `frank.wav` and return synthesised audio.

## Troubleshooting

### `could not select device driver "nvidia" with capabilities: [[gpu]]`

The NVIDIA Container Toolkit is not installed or not configured. Re-run Step 3.

---

### `exec: python: not found`

You added a `command:` override in `docker-compose.yml`. Remove it and let the image use its default entrypoint.

---

### `Failed to open the input "/app/example/frank.wav"`

The WAV file is not in the right place or the volume is not mapped correctly. Confirm:

- The file is at `speakers/frank.wav` on the host
- The volume in `docker-compose.yml` is `./speakers:/app/speakers`

---

### `no configuration file provided: not found`

You are running `docker compose up` from the wrong directory. Navigate to the folder containing `docker-compose.yml` first:

```bash
cd xtts
docker compose up -d
```

- **AI assistants** with a cloned or custom voice
- **Audiobook generation** from text or EPUB files
- **YouTube narration pipelines** — combine with faster-whisper for transcription and FFmpeg for final rendering

XTTS-v2 runs entirely on your machine, with no API costs and no data leaving your environment.
