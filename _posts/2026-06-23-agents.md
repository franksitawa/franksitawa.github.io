A chatbot answers questions. An AI agent does something more interesting — it makes decisions, uses tools, and takes a sequence of steps to complete a goal.

If LLMs are the brain, agents are what happen when you give that brain hands.

## What Makes Something an Agent?

A basic language model takes input and produces output. One turn, done.

An agent operates in a **loop**:

1. It receives a goal.
2. It decides what action to take.
3. It takes that action — searching the web, writing code, calling an API, reading a file.
4. It looks at the result.
5. It decides what to do next.

This continues until the goal is complete — or the agent gets stuck.

## A Concrete Example

Say you ask an AI agent: *"Research the top three AI tools for recruiters and summarise them in a table."*

A chatbot would try to answer from memory, probably giving you outdated information.

An agent would:
- Search the web for recent articles on AI recruitment tools
- Open and read several pages
- Compare the tools across features, pricing, and reviews
- Write a summary table based on what it actually found

Same question, very different process.

## The Tools Agents Use

Agents get access to **tools** — functions they can call to interact with the world. Common ones include:

- **Web search** — find current information
- **Code execution** — write and run code to solve problems
- **File reading/writing** — work with documents
- **API calls** — interact with external services like calendars, databases, or email

The agent decides *which* tool to use and *when*, based on the goal and what it's learned so far.

## Why This Is a Big Deal

Most real-world tasks aren't single questions — they're processes. Research, then analyse, then draft, then send. Agents can handle multi-step workflows that would otherwise require a human in the loop at every stage.

This is why agents are showing up in software engineering, customer support, and business operations — anywhere a task has multiple steps that currently need a human at every stage.

## What to Watch Out For

Agents need guardrails. An agent that can take actions can also take the *wrong* ones — deleting files, sending premature emails, or looping on a task it can't complete.

Good agent design includes human checkpoints, especially for high-stakes tasks. Autonomy and oversight work together, not against each other.
